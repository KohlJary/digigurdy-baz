144// This is the high voltage mark. It determines how easily the crank makes the drones start.
145// With my crank, I can go as low as 2, but it gets ridiculously sensitive (bumping into your gurdy
146// agitates the crank enough to register). It doesn't need to be very high, though, unless you want
147// you crank to have a "minmum speed limit" before it starts sounding.
148//
149// Especially if you are using a bridge rectifier to have a 2-way crank, you'll want this **as low as you can.**
150const int VOL_THRESHOLD = 5;
151
152// (the equivalent of V_THRESHOLD for buzzing is what the knob does, so there's no variable for it).
153
154// The crank uses an internal "spin" counter to make it continue to play through the inconsistent
155// raw voltage the crank produces. Each high voltage read increases the counter by SPIN_WEIGHT,
156// up to MAX_SPIN. While low voltage is being read, spin decreases by SPIN_DECAY. While spin is
157// greater than SPIN_THRESHOLD, it makes sound.
158//
159// A larger MAX_SPIN versus the SPIN_DECAY make the cranking more consistent at very low speeds,
160// at the expense of responding quickly when you stop cranking. A higher SPIN_WEIGHT influences
161// how quickly the crank kicks on when you start spinning, and should be at least a few times larger
162// than the SPIN_DECAY. SPIN_THRESHOLD influences how quickly the noise cuts off after you stop
163// spinning.
164const int MAX_SPIN = 7600;
165const int SPIN_WEIGHT = 2500;
166
167// I think I've got good values for everything else... in my opinion at least, users should try adjusting just
168// the SPIN_DECAY at first to change responsiveness. Lower values make notes last longer over jitters in the
169// crank voltage. Higher values make it run down spin faster overall, increasing how quickly you can
170// play staccato.
171//
172// Users using a rectifier for two-way cranking, you'll want to fiddle with this for personal feel.
173// Users using a normal crank like John does it, you'll just want to increase if your crank is jittery at low
174// speeds. You don't need to adjust it much to hear a diffrence (try +- 20 at first).
175const int SPIN_DECAY = 200;
176
177// This is the point notes start:
178const int SPIN_THRESHOLD = 5001;
179// This is the point they stop:
180const int SPIN_STOP_THRESHOLD = 1000;
181
182// Buzzing works sort of the same way except the buzz counter jumps immediately to the
183// BUZZ_SMOOTHING value and then begins to decay by BUZZ_DECAY. Any positive "buzz"
184// value makes a buzz.
185//
186// Larger BUZZ_SMOOTHING values make the buzz stay on more consistently once you've triggered it,
187// but make it harder to make separate coups rapidly. Adjust this to find the sweet spot between
188// how easy you want it to buzz and how quickly/consistently you can work a crank. Players much
189// better than me may want a smaller value. ***THE SPIN VALUES DON'T AFFECT BUZZ**
190const int BUZZ_SMOOTHING = 250;
